# Audit V1 et décisions CAP DAEU V2

## Ce qui fonctionnait déjà

- une identité personnelle claire autour du DAEU B ;
- un lancement rapide du Pomodoro ;
- des pages distinctes pour agenda, sessions, tâches, cartes et matières ;
- des données locales sans compte ;
- une base compatible GitHub Pages.

## Limites observées

1. Sur téléphone, la navigation occupait presque tout le premier écran.
2. L'agenda répétait une semaine type mais ne matérialisait pas les mois jusqu'à octobre.
3. Le conseil du jour dépendait surtout du jour de la semaine.
4. Les niveaux de matière étaient statiques.
5. Les flashcards n'avaient pas de véritable planification espacée.
6. Les erreurs, examens blancs et bilans hebdomadaires n'étaient pas reliés au plan.
7. Les statistiques décrivaient peu la progression réelle.

## Réponses appliquées dans la V2

- barre mobile fixe et menu complet secondaire ;
- prochaine action unique dans le premier écran ;
- moteur fondé sur la maîtrise, la priorité, les erreurs, les cartes dues et l'oubli ;
- plan généré jusqu'au 31 octobre 2026 ;
- agenda en mois, semaine et liste ;
- Focus 15/25/40/50 minutes, alarme, notification, distractions et bilan ;
- maîtrise dynamique par chapitre ;
- répétition espacée inspirée des principes d'Anki ;
- carnet d'erreurs avec rappels ;
- examens blancs avec notes normalisées sur 20 ;
- graphiques de temps, répartition, notes et régularité ;
- export/import JSON et PWA installable.
